Detailed Notes on Git and GitHub

1. What is Git?

Git is a distributed version control system (DVCS) used to:

* Track code changes
* Collaborate with developers
* Maintain project history
* Revert mistakes safely

Created by:
Linus Torvalds

⸻

2. What is GitHub?

GitHub is a cloud platform for:

* Hosting Git repositories
* Team collaboration
* Code review
* CI/CD integration
* Open-source development

Git = version control tool
GitHub = online hosting service for Git repositories

⸻

3. Why Git is Important

Version Tracking

Tracks every change made to files.

⸻

Collaboration

Multiple developers can work simultaneously.

⸻

Backup

Repositories can be stored remotely.

⸻

Branching

Developers can experiment safely.

⸻

Open Source Contribution

Most open-source projects use GitHub.

⸻

4. Basic Git Architecture

Git has three main areas:

Working Directory
       ↓
Staging Area
       ↓
Local Repository
       ↓
Remote Repository (GitHub)

⸻

5. Key Git Concepts

Repository (Repo)

A project tracked by Git.

Initialize:

git init

⸻

Commit

A snapshot of project changes.

⸻

Branch

An independent line of development.

Default branch:

main

⸻

Clone

Copying a remote repository locally.

git clone <repository-url>

⸻

Push

Upload local commits to GitHub.

⸻

Pull

Download latest changes from GitHub.

⸻

6. Installing Git

Official website:

Git Official Website￼

Check installation:

git --version

⸻

7. Configuring Git

Set username:

git config --global user.name "Your Name"

Set email:

git config --global user.email "your@email.com"

View config:

git config --list

⸻

8. Creating a Repository

Initialize Git

git init

Creates:

.git/

This folder stores Git metadata.

⸻

9. Git Workflow

Step 1 — Modify Files

Edit project files.

⸻

Step 2 — Add to Staging Area

git add .

OR:

git add filename.py

⸻

Step 3 — Commit Changes

git commit -m "Added login feature"

⸻

10. Checking Repository Status

git status

Shows:

* Modified files
* Staged files
* Untracked files

⸻

11. Viewing Commit History

git log

Compact view:

git log --oneline

⸻

12. Understanding the Staging Area

The staging area acts like a preparation zone before committing.

Working Directory → Staging Area → Commit

Benefits:

* Selective commits
* Organized history

⸻

13. Branching in Git

Create Branch

git branch feature-login

⸻

Switch Branch

git checkout feature-login

Modern alternative:

git switch feature-login

⸻

Create + Switch

git checkout -b feature-login

⸻

14. Merging Branches

Merge feature branch into main:

git checkout main
git merge feature-login

⸻

15. Merge Conflicts

Occurs when:

* Two branches edit the same lines

Git asks for manual resolution.

Conflict markers:

<<<<<<< HEAD
Current branch code
=======
Incoming branch code
>>>>>>> feature

⸻

16. Connecting Git to GitHub

Create repository on:
GitHub Official Website￼

Connect remote:

Verify:

git remote -v

⸻

17. Pushing Code to GitHub

First push:

Later pushes:

git push

⸻

18. Pulling Changes

git pull

Downloads and merges remote changes.

⸻

19. Cloning a Repository

22. Difference Between Git and GitHub

Git	GitHub
Version control software	Cloud hosting platform
Works locally	Works online
Tracks changes	Stores repositories
Command-line tool	Web platform

30. GitHub Authentication

Modern GitHub uses:

* Personal Access Tokens (PAT)
* SSH keys

Passwords are mostly deprecated.

⸻

31. SSH Authentication Setup

Generate key:

ssh-keygen -t ed25519 -C "your@email.com"

Add to GitHub SSH settings.

Test:

ssh -T git@github.com

34. Common Mistakes Beginners Make

Uploading venv/

Avoid uploading virtual environments.

⸻

Committing Secrets

Never commit:

* API keys
* Passwords
* .env files

⸻

Large Binary Files

Git is optimized for code, not huge media files.

⸻

35. Best Practices

Write Meaningful Commit Messages

Good:

Added JWT authentication middleware

Bad:

fixed stuff

⸻

Commit Frequently

Small commits are easier to manage.

⸻

Pull Before Push

Avoid merge conflicts.

⸻

Use Branches

Never develop directly on main.
